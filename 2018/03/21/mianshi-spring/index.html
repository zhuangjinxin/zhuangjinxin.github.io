Spring

Spring是一套轻量级的、企业级的Java开发框架。

AOP

概念：
AOP 面向切面编程，把多个业务模块的公共行为封装成一个可重用的模块，通过动态代理的方式让业务模块具备这些功能；增加代码复用率，降低代码耦合度。

应用：
日志模块、事务管理、权限管理等

实现原理：
动态代理
动态代理的实现方法：
实现InvocationHandler接口，重写invoke方法；在Invoke方法里写增强处理的逻辑，这个公共代理类在运行的时候才能明确自己要代理的对象，同时可以实现该被代理类的方法的实现，然后在实现类方法的时候可以进行增强处理。

AOP实现的方式：
1.通过注解 @Aspect定义切面 @Before/@After/@Around(表达式) 在切入方法执行前后插入功能
2.通过Spring的API实现AOP
3.自定义类来实现AOP，不实现spring的自带的通知

用到的Jar包：
spring-aop.jar aopalliance.jar aspectsweaver.jar spring-aspects.jar

IoC

概念:

实现原理:
依赖注入DI
依赖注入的方法：setter注入、注解注入、构造器注入

Spring的事务

事务的传播特性:
REQUIRED:（默认）如果当前存在事务，则加入该事务；如果当前没有事务，则创建一个新的事务；
REQUIRES_NEW: 创建一个新的事务，如果当前存在事务，则把当前事务挂起；
SUPPORTS: 如果当前存在事务，则加入该事务；如果当前没有事务，则以非事务的方式运行；
NOT_SUPPORTED: 以非事务方式运行，如果当前存在事务，则把当前事务挂起；
NEVER: 以非事务方式运行，如果当前存在事务，则抛出异常；
MANDATORY: 如果当前存在事务，则加入该事务；如果当前没有事务，则抛出异常；
NESTED: 如果当前存在事务，则创建一个事务作为当前事务的嵌套事务来运行；如果当前没有事务，则该取值等价于REQUIRED；

事务的隔离级别
DEFAULT: 默认值 表示使用数据库的默认隔离级别。对于大多数数据库，该值等价于READ_COMMITTED；
READ_UNCOMMITTED: 读未提交 表示一个事务可以读取另一个事务修改但还没有提交的数据；不能防止脏读、不可重复读和幻读；
READ_COMMITTED: 读已提交 表示一个事务只能读取另一个事务已经提交的数据；可以防止脏读；
REPEATABLE_READ: 可重复读 表示一个事务在整个过程中可以重复执行某个查询，并且每次返回的记录都相同；可以防止脏读和不可重复读；
SERIALIZABLE: 可串行化 表示所有的事务依次逐个执行；可以防止脏读、不可重复读和幻读；

Spring MVC的请求处理过程

DispatcherServlet根据逻辑视图名找到View Resolver，并返回时图对象；

BeanFactory和ApplicationContext的关系

ApplicationContext是BeanFactory的字接口，BeanFactory提供了Spring的配置框架和基本功能，ApplicationContext则添加了更多企业级功能（如国际化的支持）；
ApplicationContext容器初始化完成后，容器中所有的singleton Bean 也都被实例化了，使用singleton Bean时无需等待；
其他BeanFactory接口的实现类，会延迟到调用getBean（）方法时构造；资源较少的情况下可以使用。
