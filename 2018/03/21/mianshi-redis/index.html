面试汇总:《Redis》

发表于 2018-03-21 更新于 2018-12-15 分类于 笔试面试 阅读次数：

为什么Redis性能高？

纯内存存储、I/O多路复用(epoll)、单线程架构

Redis常用数据类型

字符串、哈希、列表、集合、有序集合

常用命令

string: set get mset mget del incr decr append strlen setrange getrange
hash: hset hget hmset hmget hdel hgetall hexists hkeys hvalues hsetnx hincrby hstrlen
list: lpush rpush lpop rpop blpop brpop lrange lindex llen lset
set: sadd srem scard smermbers sismember srandmember spop|sinter sunion sdiff
zset: zadd zrem zcard zincrby zrank zrevrank zrange zrangebyscore zinterstore zunionstore
发布订阅: publish subscribe unsubscribe psubscribe pubsub
键管理: type object exists dbsize rename renamenx randomkey
过期时间: expire expireat pexpire pexpireat ttl pttl persist
迁移: move dump+restore migrate
遍历键: keys scan
数据库: select flushdb flushall

注意事项

  • scan命令可以解决keys命令可能带来的阻塞问题；
  • set命令会删除字符串类型键的过期时间；
  • 批量操作可以提高命令执行效率(mset/mget/hmset)；

Redis的应用

  • 实现分布式锁: setnx
  • 实现栈(Stack): lpush + lpop
  • 实现队列(Queue): lpush + rpop
  • 实现消息队列(MQ): lpush + brpop
  • 给用户打标签(Tag): sadd
  • 生成随机数/抽奖: srandmember/spop
  • 实现社交需求：sadd + sinter
  • 实现排行榜/点赞数：sortedset
  • 实现延迟队列: sortedset（score时间戳）

Redis持久化机制

Redis支持RDB和AOF两种持久化机制，持久化可以避免因进程退出造成数据的丢失问题，当下次重启时利用持久化的文件即可实现数据恢复。

RDB：fork操作创建子进程把内存中的数据以快照的方式写入二进制文件中 dump.rdb
AOF：把每一个收到的写命令都通过write函数追加到文件中（appendonly.aof）,bgrewriteaof压缩aof的持久化文件

数据淘汰策略

volatile-lru: 从设置了过期时间的数据中，删除最近最少使用的数据；
allkeys-lru：从全部数据中，删除最近最少使用的数据；
volatile-random：从设置了过期时间的数据中，随机删除一些数据；
allkeys-random：从全部数据中，随机删除一些数据；
volatile-ttl：从设置了过期时间的数据中，删除即将要过期的数据；
noeviction：(默认)不删除任何数据，直接返回错误；

LRU：Less Recently Used，最近最少使用
LFU：Less Frequently Used，最不频繁使用（@since redis 4.0）
TTL：Time To Live，存活时间

Redis常见故障

缓存穿透：
问题：查询一个缓存中一直不存在的数据，导致每次请求都查询DB。在流量比较大时，DB可能会挂掉。
办法：把不存在的key预设一个特殊值，并设置过期时间。

缓存雪崩：
问题：同一时间很多缓存过期，导致大量的请求查DB。
办法：分散缓存过期时间

缓存并发：
问题：并发高时，如果一个缓存失效，可能会有大量请求同时查询DB。
办法：查询加锁，获取锁的进程查DB并写入缓存。
