面试汇总:《MySQL》

发表于 2018-03-21 更新于 2018-12-15 分类于 笔试面试 阅读次数：

数据库原理

三大范式

第一范式： 表中的每一列都是不可拆分的最小单元
第二范式： 表中的每一列必须依赖于主键，不能与主键没有关系
第三范式： 每个非主属性都必须与主键直接相关，而不间接相关

五大约束

PRIMARY KEY： 主键约束
UNIQUE： 唯一性约束
DEFAULT： 默认值
NOT NULL： 非空约束
FOREIGN KEY： 外键约束

数据库事务

1).四大特性 ACID
原子性、一致性、隔离性、持久性

2).隔离级别
Read Uncommitted 读未提交
Read Committed 读已提交
Repeatable Read 可重复读
Serializable 串行化

3).脏读、不可重复读、幻读
脏读
不可重复读
幻读

SQL注入

1).什么是SQL注入？
把SQL命令插入到表单中提交，最终达到欺骗服务器执行恶意的SQL命令。

2).防止SQL注入的办法：
校验用户输入的内容
参数化SQL语句
权限限制

MySQL

InnoDB与MyISAM的区别

1.InnoDB支持行级锁、事务、外键，MyISAM不支持；
2.读多写少时，MyISAM；读少写多时，用InnoDB；

索引

1).索引的类型
普通索引、唯一索引、主键索引、组合索引、全文索引

2).索引的数据结构
B+Tree

3).组合索引的特点
最左前缀

4).索引算法
二分查找法

锁
事务

1).事务概念
(同数据库部分）略

2).默认隔离级别
MySQL：Repeatable Read 可重复读
Oracle：Read Committed 读已提交

3).查询和修改事务隔离级别
查询：select @@tx_isolation;
修改：set global transaction isolation level repeatable read;

MyISAM和InnoDB的区别

InnoDB支持行级锁、外键，事务，MyISAM不支持；
MyISAM支持全文索引，InnoDB不支持；
读多写少，MyISAM快；写多读少，InnoDB快；

SQL调优

1).慢查询
EXPLAIN 查看执行计划

2).分析流程
1.查看数据库状态 show state
2.开始慢查询日志 my.cnf slow query log
3.分析SQL语句 explain执行计划 profiling消耗资源

3).常用优化方法
1.查询优化
不要select *
小表驱动大表
使用连接代替子查询
适当冗余数据 减少连接

2.索引优化
哪些列适合索引、哪些不适合
索引创建和使用规则
索引失效

3.数据库设计
合适的数据类型
拆表
读写分离

4.硬件调优

集群

1).主从复制
1.作用
读写分离，提供查询服务
实时灾备，用于故障切换
备份，避免影响业务

2).复制过程

  1. I/O线程读取master的binlog，写入Relaylog(中继日志)；
  2. SQL线程读取并解析Relaylog，实现主从操作一致

3).常见问题

  1. 主库宕机，数据丢失(半同步复制)
    • 确保事务提交后binlog至少传输到一个从库(ACK)
    • 不保证从库应用完这个事务的binlog
  2. 从库只有一个SQL线程，复制延迟(并行复制)
    • 并行是指从库多线程apply binlog
    • 库级别并行应用binlog，同一个库数据更改还是串行的

4).监控
show slave status 查看主从同步状态

5). 读写分离
