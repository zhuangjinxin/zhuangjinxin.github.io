面试汇总:《Java》

发表于 2018-03-21 | 分类于 面试 | 阅读次数

集合

Java中常用的数据结构

List: ArrayList、LinkedList
Set: HashSet
Map: HashMap、ConcurrentHashMap

ArrayList、LinkedList

ArrayList：基于动态数组实现 默认大小10 扩容1.5倍+1 线程不安全 CopyOnWriteArrayList或Vector
LinkedList：基于双向链表实现
区别：
1.随机访问元素时，ArrayList优于LinkedList；(因为LinkedList要移动指针)
2.新增或删除元素时，LinkedList优于ArrayList；（因为ArrayList要移动数据）
注意：Arraylist不能用正向for循环删除元素

HashMap、ConcurrentHashMap、HashTable

HashMap：

基于数组+链表的方式实现

  1. 初始化与扩容机制
    初始化默认大小为16，扩容因子(加载因子)为0.75；当map中存储数据的容量大于当前map的容量*加载因子时，把map容量扩大1倍。

  2. put和get操作的大致流程

    • put：
      1.对key进行hashcode操作，计算出index；
      2.如果没有碰撞，直接放入；
      3.如果发生碰撞，以链表的形式追加到index元素后链表的末端；
      .如果碰过导致链表过长，把链表转化成红黑树；
      5.如果key已经存在，覆盖掉旧值；
      6.如果存储数据达到一定个数，则扩容；

    • get：
      1.对key进行hashcode操作，计算出index；
      2.如果index位置只有一个元素，直接取出；
      3.如果index位置是链表或树，通过key.equals(k)查找；(复杂度 链表O(n) 树O(log n))

ConcurrentHashMap：

锁分段 (JDK1.8 CAS) @since1.5 数组+单向链表o(n)+红黑树o(logN) 锁的粒度小，效率高

HashTable：

使用Synchronized保证线程安全 效率低

区别：

  1. HashMap线程不安全、 HashTable、ConcurrentHashMap线程安全
  2. HashMap key或Value可以为空,HashTable和ConcurrentHashMap key和value都不能为空；

工具类Collections、Arrays

常用方法：排序、数组和列表转化、二分查找、反转

多线程

线程的状态及转换流程

新建、就绪、运行、阻塞、死亡

实现多线程的方式

  1. 继承Thread类
  2. 实现Runnable接口
  3. 实现Callable接口(通过FeatureTask包装器创建线程)
  4. 使用ExcutorService、Callable、Future接口实现有返回结果的线程

    ThreadPoolExecutor

  • corePoolSize 核心线程池大小
  • maximumPoolSize 最大线程池大小
  • keepAliveTime 大于核心线程池数量的空闲线程最大存活时间
  • unit 时间单位
  • BlockingQueue 阻塞队列(ArrayBlockingQueue、LinkedBlockingQueue、SynchronousQueue)
  • ThreadFactory 线程工厂
  • RejectedExecutionHandler 任务的拒绝策略（AbortPolicy、DiscardPolicy、DiscardOldestPolicy、CallerRunsPolicy）

Excutors创建线程池的方式

  • newFixedThreadPool() 创建固定大小的线程池
  • newSingleThreadExecutor() 创建只有1个线程的线程池
  • newCachedThreadPool() 创建带缓存的线程池
  • newScheduledThreadPool() 创建定时的线程池

线程间的通信方式

  • 共享变量(CountDownLatch)
  • 信号量
  • 管道
  • wait/notify
  • lock/condition

向线程池提交任务的方式及区别

  • excute() 无返回值
  • submit() 有返回值
    • 通过Future.get()获取返回值，一直阻塞直到任务完成；
    • Future.get(million,unit) 设置超时时长，防止一直阻塞;

线程的关闭方法

  • shutdown 将线程池的状态置为SHUTDOWN，中断没有正在执行的任务；
  • shutdownNow 将线程池的状态置为STOP，调用interrupt中断正在执行的线程，返回等待任务的列表。

死锁

死锁的概念

死锁的必要条件

  • 互斥使用
  • 不可抢占
  • 请求和保持
  • 循环等待

实现死锁

消除死锁

  • synchronized
  • Lock显示锁(Lock.unlock())

避免死锁

  1. 避免一个线程同时获取多个锁
  2. 避免一个线程在锁内同时占用多个资源，尽量保证每个锁只占用一个资源
  3. 尝试使用定时锁，使用lock.tryLock来代替使用内置锁

JVM

JVM的内存模型

  • 堆 Heap ：存放对象实例
  • 方法区 Method Area：存储类加载信息、常量、静态变量等
  • 虚拟栈 VM Stack：存放局部变量表、操作栈、动态链接、方法出口
  • 本地方法栈 Native Method Stack
  • 程序计数器 Program counter Register：存放线程执行的字节码的行号

垃圾回收机制

垃圾回收方式

  1. 标记-清除法 Mark-Sweep：
    概念：先标记需要回收的对象，标记完成后统一回收。
    缺陷：1.标记和清理过程效率都不高；2.产生大量不连续的内存碎片；

  2. 复制 Copying：
    概念：划分内存区块，每次把还存活的对象从一块区域复制到另一块区域，原区域直接清除。
    优点：1.每次可以对一块区域整体回收，内存分配不必考虑内存碎片问题；2.只需要移动堆顶指针，按顺序分配内存即可，实现简单、运行高效；
    缺点：1.当对象存活率较高时需要进行大量的复制操作，效率会变低；2.不使用对分策略，需要有额外的空间进行分配担保；

  3. 标记-整理法 Mark-Compact：
    概念：先标记可回收的对象，把所有存活的对象向一端移动，然后清理掉端边界以外的内存。

  4. 分代收集法 Generational Collecting：
    概念：根据对象存活周期不同将内存划分为年轻代和老年代，年轻代采用复制法回收，老年代通过标记-清除法或标记-整理法回收。
    具体：Java堆：

  • 年轻代 Young Generation
    • Eden
    • Survivor 0(From区)
    • Survivor 1(To区)
      每次回收Eden和其中的一个Survivor区中还存活的对象复制到另一个Survivor区，然后清楚Eden区和刚才用过的那个Survivor区；
  • 老年代 Old Generation
    存放N次垃圾回收之后还存活的对象；老年代存满时触发FULL GC。
  • 持久代 Permanent Generation
    用于存储class类、常量、方法描述等。主要回收废弃常量和无用的类。持久代对垃圾回收没有显著影响。

设置JVM常用的参数

堆设置(常用 重点)

  • -Xms : 初始堆大小 默认堆内存的1/64
  • -Xmx : 最大堆大小 默认堆内存的1/4
  • -Xmn : 年轻代大小 默认堆内存的1/64 sun推荐配置堆内存的3/8
  • -XX:NewSize=n : 年轻代大小
  • -XX:NewRatio=n : 年老代和年轻代比值 默认2:1(n=2)
  • -XX:SurvivorRatio=n : 年轻代中Eden区与两个Survivor区的比值 默认8:1:1(n=8)
  • -XX:MaxPermSize=n : 持久代大小

收集器设置

  • -XX:+UseSerialGC : 设置串行收集器
  • -XX:+UseParallelGC : 设置并行收集器
  • -XX:+UseParalledlOldGC : 设置并行年老代收集器
  • -XX:+UseConcMarkSweepGC : 设置并发收集器

垃圾回收统计信息

  • -XX:+PrintGC:开启GC简要日志模式
  • -XX:+PrintGCDetails:开启GC详细日志模式
  • -XX:+PrintGCTimeStamps:打印GC时间(相对JVM启动的时间)
  • -XX:+PrintGCDateStamps:打印GC时间(真实时间)
  • -Xloggc:filename:GC日志输出到文件

并行收集器设置

  • -XX:ParallelGCThreads=n:设置并行收集器收集时使用的CPU数.并行收集线程数
  • -XX:MaxGCPauseMillis=n:设置并行收集最大暂停时间
  • -XX:GCTimeRatio=n:设置垃圾回收时间占程序运行时间的百分比.公式为1/(1+n)

并发收集器设置

  • -XX:+CMSIncrementalMode:设置为增量模式.适用于单CPU情况
  • -XX:ParallelGCThreads=n:设置并发收集器年轻代收集方式为并行收集时,使用的CPU数.并行收集线程数

Java指令重排

为什么要指令重排？

JVM为了优化指令，提高程序的运行效率；在不影响单线程运行结果的前提下，尽可能的提高并行度。
包括编译器重排和运行时重排

指令重排会导致哪些问题？

  • 一个线程指令重排导致另一个线程出错
  • 导致单例模式失效

如何避免指令重排？

使用Volatile关键字禁止指令重排
拓展：Volatile通过内存屏障的方式防止指令重排，内存屏障插入策略：

  • 写操作前，加StoreStore屏障；
  • 写操作后，加StoreLoad屏障；
  • 读操作前，加LoadLoad屏障；
  • 读操作后，加LoadStore屏障；

Volatile关键字的作用

  • 保证内存可见性
  • 防止指令重排
