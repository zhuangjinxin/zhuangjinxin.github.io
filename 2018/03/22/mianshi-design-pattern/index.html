面试汇总:《设计模式》

发表于 2018-03-22 | 分类于 面试 | 阅读次数

设计模式代表了最佳的实践,是一套被反复使用、多数人知晓、经过分类编目、代码设计经验的总结。使用设计模式是为了重用代码、让代码更容易被他人理解、保证代码可靠性。

常用的设计模式
创建型模式: 单例模式、工厂模式、抽象工厂模式、建造者模式、原型模式
结构型模式: 适配器模式、装饰器模式、代理模式、外观模式、桥接模式、组合模式、享元模式
行为型模式: 模版方法模式、策略模式、观察者模式、中介者模式、访问者模式、责任链模式、迭代器模式、解释器模式、命令模式、备忘录模式、状态模式

注: 只回答最熟悉的设计模式，包括但不限于: 单例模式、工厂模式、装饰器模式、适配器模式、代理模式、观察者模式、模版方法模式

单例模式

定义：保证一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。
解决了什么问题：解决一个类频繁的创建和销毁
实现方式：懒汉式、饿汉式、双检锁、登记式、枚举

懒汉式

延迟初始化(懒加载)
说明: 去掉synchronized，线程不安全；加上synchronized，线程安全；

饿汉式

不是懒加载
线程安全
不加锁，执行效率高

双检锁

双重校验锁 DCL
懒加载
线程安全
双锁机制
@since JDK1.5

登记式

静态内部类
懒加载
线程安全
适用于静态域(如果是实例域，用双检锁)

枚举

不是懒加载
线程安全
自动支持序列化机制，防止反序列化重新创建新对象
@since JDK1.5

工厂模式

定义: 定义一个创建对象的接口，让子类决定实例化哪一个工厂类，工厂方法使一个类的实例化延迟到其子类。
解决什么问题: 解决接口选择的问题
应用场景: 选择支付方式、选择供应商、选择发送消息类型

工厂模式分类: 普通工厂模式、多个工厂方法模式、静态工厂方法模式

建造者模式

定义: 把复杂对象的构建和表示分离，同样的构建过程创建不同的表示。
解决了什么问题：解决了复杂对象的创建问题

原型模式

定义: 通过拷贝原型实例创建新对象
实现方式：1.实现Cloneable接口；2.重写Object类中的clone方法；

适配器模式

定义：
解决了什么问题：解决两个接口的兼容问题
应用场景:

  • 类的适配器模式：当希望把一个类转换成满足另一个新接口的类，可以创建一个新类，继承原有的类，实现新的接口即可。
  • 对象的适配器模式：当希望将一个对象转换成满足另一个新接口的对象时，可以创建一个Wrapper类，持有原类的一个实例，在Wrapper类的方法中，调用实例的方法即可。
  • 接口的适配器模式：当不希望实现一个接口中所有的方法时，可以创建一个抽象类Wrapper，实现所有方法，我们写别的类的时候，继承抽象类即可。

装饰器模式

定义：给一个对象动态的添加新功能，装饰对象与被装饰对象实现同一个接口，同时装饰对象持有被装饰对象的实例；
应用场景：1.扩展一个类的功能；2.动态的增加或撤销一个对象的功能；

代理模式

定义：添加一个代理类代替原对象进行一些操作
应用场景：1.修改原有的方法来适应需求；2.对原对象产生的结果进行控制；

外观模式

解决了什么问题：解决了类与类之间的耦合关系

桥接模式

应用场景：数据库中通过DriveManager管理MySQL、Oracle的连接

组合模式

享元模式

模版方法模式

定义: 把公共的方法抽象出来放在抽象类中作为模版，在子类中重写这些方法具体实现；
结构: 一个抽象类 + 一组(个)实现类
解决了什么问题：一些方法通用，却在每个子类都重新写了这些方法
应用场景：多个子类中有相同的方法、主框架相同细节不同

策略模式

定义: 定义一组算法，将每个算法都封装起来，并且使他们之间可以互换。
结构：Context上下文 + 抽象策略类 + 具体策略类
解决什么问题：避免了在多种相似算法的情况下因使用if…else带来的复杂性

观察者模式

定义: 当一个对象改变状态时，所有依赖于它的对象都会得到通知并自动更新。
结构: 观察者接口 + 观察者实现 + 被观察者接口 + 被观察者实现
应用场景：Java中Observer接口及它的实现类

中介者模式

定义: 用中介者对象封装和处理对象之间的交互，同事类之间不再需要显示地交互。
结构：抽象中介者 + 多个同事类
解决了什么问题：解决了同事类之间复杂的交互网络，降低同事类间的耦合度

访问者模式

定义: 访问者类中封装对元素的某些操作，可以在不改变元素结构的情况下，通过修改访问者类的方法来操作元素；
结构：抽象访问者 + 访问者 + 抽象元素类 + 元素类 + 结构对象
解决了什么问题：解决了稳定数据结构和易变的操作之间的耦合问题

责任链模式

定义: 把多个对象连成一条链，沿着链传递请求，直到有对象处理它。
做法: 把条件判断的逻辑放到各个处理类中，客户端可以随意设定处理顺序；
结构: 一个抽象处理类 + 一组具体处理类
解决了什么问题：降低耦合度

迭代器模式

定义: 提供一种遍历容器中元素的方法，而不暴露该容器对象的内部细节。
应用场景: Iterator遍历集合元素(List、Set、Map、数组)

解释器模式

定义: 给一种语言定义文法、定义解释器，用解释器解释文法。
结构：抽象解释器 + 终结符表达式 + 非终结符表达式 + 环境角色
应用场景:重复的加减乘除运算

命令模式

定义: 将请求封装成对象，可以用不同的请求把客户端参数化。
结构: 命令抽象类 + 命令实现类 + 客户端调用类 + 调用者 + 接收者
做法: 把调用者和执行者分开，不必关心对方如何操作
应用场景：实现请求-响应功能

备忘录模式

定义: 捕获对象的内部状态，并在该对象之外保存这个状态，以便回滚。
结构：发起人 + 备忘录 + 管理角色
解决了什么问题：对象状态回滚
